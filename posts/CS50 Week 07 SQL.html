Lambda functions are similar to user-defined functions but without a name. They're commonly referred to as anonymous functions. Lambda functions are efficient whenever you want to create a function that will only contain simple expressions – that is, expressions that are usually a single line of a statement.

Stores all of the data in columns and rows.

You can do things in SQL in one line but in another language like Python you will need to write multiple lines to perform the same task, SQL is more powerful at dealing with data.

One to one relationship

In a one-to-one relationship, one record in a table is associated with one and only one record in another table. For example, in a school database, each student has only one student ID, and each student ID is assigned to only one person.

One to many relationship

In a one-to-many relationship, one record in a table can be associated with one or more records in another table. For example, each customer can have many sales orders. In this example the primary key field in the Customers table, Customer ID, is designed to contain unique values.

Primary Key and Foreign Key

A Foreign Key is a key used to link two tables. The table with the Foreign Key Constraint (aka “child table”) is connected to another table (aka, the “parent table”). The connection is between the child table’s Foreign Key Constraint and the parent table’s Primary Key.

Instead of duplicating data all over your database you can use foreign keys to refer to data in another table, for example let's say you have a store database, in one table your have the products, in another you have the sales, you can use foreign keys to refer to the products in the other table instead of duplicating the products names.

You can nest SQL commands, for example you can run a command on the returned value from another command.

B-Tree is like a binary tree, but a node can have more than 2 children nodes.

Indexes

Stores a chunk of the database in memory for faster access.

SQL injection attacks, when you wrote SQL code as input for example in a web form you can change the original SQL command to something else.

That's why you should avoid using formatted strings when take input from user to complete a SQL command. Instead pass the input as an argument and use the ? mark because SQL libraries will ignore and treat those injections as normal text.
