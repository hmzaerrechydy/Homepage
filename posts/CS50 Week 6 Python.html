floating point imprecision

One way of understanding "why" is to realize that there are an infinite number of floating point numbers. But you are storing it into a finite number of bits in memory. So the number of different numbers you can store is finite; some approximation is going to have to be made for some numbers.

There's no more integer overflow because Python will give you more bits as you need them.

strings are immutable in python

In Python, strings are made immutable so that programmers cannot alter the contents of the object but you can get a copy, but you can't change the original string in memory.
